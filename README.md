# Fictional IT Recruitment Analysis & Semantic Search

This project evaluates synthetic recruitment data for a fictional IT company. It uses NLP and machine learning techniques to assess candidate CVs and predict recruitment outcomes (Accepted or Rejected).

## Project Overview

1. **Dataset**: 10,000 synthetic recruitment records generated for various IT positions spanning from 2020 to 2026. The dataset contains job descriptions, requirements, candidate CVs, experience years, and recruitment results.
2. **Feature Extraction**: Candidate CVs are transformed into dense vector embeddings using `sentence-transformers/all-MiniLM-L6-v2`.
3. **Classification**: Multiple machine learning models are trained on the embeddings to predict the recruitment result.
4. **Application**: A Gradio web interface allows users to input a new CV, predict the likelihood of acceptance, and find the top 3 most similar historical candidates using cosine similarity.

## Model Selection Process

To ensure we use the best model for predicting recruitment outcomes based on CV embeddings, we trained and evaluated three different classifier types:

1. **Random Forest**: We utilized a Random Forest classifier (100 estimators) to establish a strong baseline. This ensemble method is excellent for providing comprehensive explainability and feature importance, which helps understand *why* the model makes certain decisions.
2. **Support Vector Machine (SVM)**: A linear kernel SVM was trained. SVMs often perform exceptionally well in high-dimensional spaces, such as the 384-dimensional dense vectors generated by the embedding model. This also provides strong theoretical guarantees and explainability through support vectors.
3. **PyTorch Neural Network (MLP)**: We implemented a Multi-Layer Perceptron using PyTorch with a hidden layer and dropout. Neural networks can capture complex non-linear relationships in dense embeddings.

### Evaluation & Results
All models were evaluated using an 80/20 train-test split (8,000 training samples, 2,000 test samples). The primary metric for model selection was the **F1-Score**, as it balances precision and recall, which is crucial for recruitment to minimize both false positives (bad hires) and false negatives (missing good candidates).

*Results based on the evaluation:*
- **Random Forest**: Accuracy ~76.7%, F1-Score ~76.6%
- **Support Vector Machine**: Accuracy ~77.4%, F1-Score ~78.1%
- **PyTorch Neural Network**: Accuracy ~80.8%, F1-Score ~78.5%

**Conclusion**: The PyTorch Neural Network achieved the highest F1-Score and Accuracy. Therefore, it was selected as the default model running in the Gradio application. The Random Forest and SVM models are also saved in the repository to allow for further comprehensive analysis of the generated results.
